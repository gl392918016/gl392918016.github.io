使用hexo构建博客
1、安装>node.js<和>git<。


2、新建一个文件blog（名字自取），在文件中右键打开git bash。

3、输入npm install -g hexo-cli,安装>hexo<。
$ npm install -g hexo-cli
4、输入hexo init，初始化该文件夹。
$ hexo init
5、输入hexo serve，开启本地服务器，（如果端口被占用，输入hexo serve -p 端口号，改变端口）。
$ hexo serve
$ hexo serve -p 5000

这时当你浏览器输入显示的本地地址，就会出现hexo默认的主题页面。

6、改变主题，进入>hexo的主题<，选择你喜欢的一款主题，点击进去后复制项目地址，输入git clone 项目地址 themes/项目名称，克隆下来主题，并在文件根目录的_config.yml文件里边找到theme:位置改变为项目名称。这里粘上>我的主题<
$ git clone https://github.com/miccall/hexo-theme-Mic_Theme.git themes/mic
7、根据主题的内容，进行更改配置和创建文章。
  • 7-1、创建博客，输入命令后会在根目录下的source文件下的_posts文件里找到新创建的文件。
    $ hexo new post '文件名称'

  • 7-2、然后编写创建好的.md文件，md文件编写不会>点击这里<

  • 7-3、改变代码高亮>hexo-prism-plugin<插件，安装命令npm i -s hexo-prism-plugin。

    $ npm i -s hexo-prism-plugin

  • 7-4、然后修改根目录下的_config.yml文件中的highlight.enable的值为false，并新增prism插件相关的配置

    prism_plugin:
mode: 'preprocess'    # realtime/preprocess
theme: 'tomorrow'
line_number: false    # default false
custom_css:

  • 7-5、图片的存储，我放在七牛云里

现在我们已经把自己想要的页面已经做好了，接下来就是部署到github上了。

8、github账号注册，然后新建一个项目，下图，我是建过了所以报错。需要注意画框两个的地方必须一致，勾选Initialize。由此可以看出一个账号只能创建一个服务。


现在查看项目的设置（导航末尾有个Settings选项），向下拉到GitHub Pages时，有个地址访问就可以发现项目已经部署到网络上了。

接下来需要将hexo项目部署到github上。

9、如果是第一次使用github你需要做：
  • 9-1、配置用户名和邮箱
    $ git config --global user.name '你的用户名'
$ git config --global user.name '你的邮箱'
  • 9-2、生成密钥，输入命令，连续回车，完成后会在你的C:\Users\Administrator\（默认地址）目录下生成一个.ssh的文件。里面有id_rsa和id_rsa.pub两个文件
    $ ssh-keygen -t rsa -C '你的邮箱'

  • 9-3、复制id_rsa.pub的内容到：登录github，点击右上角的头像，有Settings的选项点击，然后依次操作。


  • 9-4、测试是否添加成功,出现下图红框里的内容表示成功。下面有个警告需要在hosts文件里配置ip地址。如果配置失败那就删除在来一遍吧。

    $ ssh -T git@github.com

    10、如果已经配置过了，跳过上面步骤。接下来你需要在项目中配置Deployment,在根目录下找到_config.yml文件，做如下修改。

    11、然后需要安装一个扩展
    $ npm install hexo-deployer-git --save
    12、最后输入命令,生成及部署。下面命令是简写，所有命令可以>查看这里<
    $ hexo d -g
